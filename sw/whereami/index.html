How it works

I 've implemented a simple Perl script that runs within a CGI (or preferebly mod_perl) web server and accepts plain NMEA input that looks line this: 

	$GPGGA,163843.060,3759.588,N,02348.412,E,0,00,0.0,302.100,M,0.0,M,,*67
	$GPVTG,0.000,T,0,M,0.000,N,0.000,K*50
	$GPRMC,163844.070,V,3759.588,N,02348.412,E,0.00,0.00,081108,,*18

The script processes GPGGA (location update) and GPRMC (movement update) lines and logs the output to an SQLite database. Alternatively, it can directly receive and display preprocessed data from a client running in the mobile phone. The exact request URL format is the following: 

	http://cartman.dmst.aueb.gr/perl/loggps-db.pl?lat=39.62365&lon=22.409113333333334&alt=144.6&speed=0&clientid=gousiosg

I 've also implemented a small J2ME application for the mobile phone that reads data from the GPS receiver and submits it to the logging script. You can find the client program here.

After the data is in the database, displaying it on a map is a breeze with the excellent Google Maps API. Basically, a cron script periodically update the values of some javascript variables which are then used by the map as input values.
